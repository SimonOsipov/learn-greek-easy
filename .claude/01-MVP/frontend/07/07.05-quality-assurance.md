### Script 1: File Path Validator

```bash
#!/bin/bash
# validate-file-paths.sh

echo "Validating component file paths..."

FRONTEND_DIR="/Users/samosipov/Downloads/learn-greek-easy/learn-greek-easy-frontend"
DOC_FILE="/Users/samosipov/Downloads/learn-greek-easy/.claude/01-MVP/frontend/Components-Reference.md"

# Extract file paths from documentation
grep -E "^\*\*File\*\*:|^File:" "$DOC_FILE" | \
  sed 's/.*: `//' | \
  sed 's/`//' | \
while read -r filepath; do
  # Remove leading slash and add to frontend dir
  fullpath="$FRONTEND_DIR${filepath}"

  if [ -f "$fullpath" ]; then
    echo "‚úÖ $filepath"
  else
    echo "‚ùå NOT FOUND: $filepath"
  fi
done

echo "Validation complete."
```

## Common Issues and Fixes

### Issue 1: Broken File Path
**Symptom**: File path in documentation doesn't match actual file location
**Fix**: Update file path in documentation to match actual location

### Issue 2: Broken Anchor Link
**Symptom**: Internal link `[Component](#component)` doesn't work
**Fix**: Ensure anchor matches heading ID (lowercase, hyphenated)
```markdown
‚ùå [ProtectedRoute](#ProtectedRoute)
‚úÖ [ProtectedRoute](#protectedroute)
```

### Issue 3: Missing Cross-Reference
**Symptom**: Component mentions another component but doesn't link to it
**Fix**: Add link in "Related Components" section
```markdown
**Related Components**:
- [OtherComponent](#othercomponent) - Description of relationship
```

### Issue 4: Incomplete Interface
**Symptom**: TypeScript interface missing props used in examples
**Fix**: Add missing props to interface definition

### Issue 5: Inconsistent Formatting
**Symptom**: Some components use different heading levels or table formats
**Fix**: Standardize formatting to match existing documented components

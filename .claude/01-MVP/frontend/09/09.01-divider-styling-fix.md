## Code Reference

### Complete Divider Pattern

This pattern is commonly used in authentication forms and follows Tailwind CSS best practices:

```tsx
{/* OAuth Divider - Text centered between horizontal lines */}
<div className="relative my-6">
  {/* Full-width horizontal line */}
  <div className="absolute inset-0 flex items-center">
    <div className="w-full border-t border-gray-300"></div>
  </div>

  {/* Centered text with white background to "cut" the line */}
  <div className="relative flex justify-center text-sm">
    <span className="bg-white px-4 text-gray-500">OR CONTINUE WITH</span>
  </div>
</div>
```

**How it works**:
1. **Outer container** (`relative my-6`): Establishes positioning context and vertical spacing
2. **Absolute line** (`absolute inset-0`): Creates full-width horizontal line behind text
3. **Centered text container** (`relative flex justify-center`): Centers text horizontally
4. **Text with background** (`bg-white px-4`): White background "cuts" the line, padding creates gap

**Why this pattern**:
- Standard Tailwind CSS approach for divided sections
- No custom CSS needed
- Responsive by default
- Easy to maintain and understand

**Rejected**: Requires more markup, harder to center perfectly

## A. Task Overview

### Purpose
Install and configure the Shadcn/ui component library to provide a robust foundation of accessible, customizable React components that integrate seamlessly with our Tailwind CSS configuration.

### Why Shadcn/ui?
- **Accessible by Default**: Built on Radix UI primitives with full ARIA support
- **Customizable**: Components are copied to your codebase, not imported from node_modules
- **Tailwind Native**: Designed specifically for Tailwind CSS projects
- **TypeScript First**: Full TypeScript support out of the box
- **Tree-Shakeable**: Only ship the components you actually use
- **No Runtime Dependencies**: Components become part of your codebase

### Integration Requirements
- Must inherit and work with our existing custom Tailwind theme
- Components must respect our Style Guide color palette
- All 14+ required components from Components Reference must be installed
- Path aliases must be configured for clean imports

## C. Step-by-Step Instructions

### Step 1: Install Required Dependencies for Shadcn/ui

Before initializing Shadcn/ui, we need to install some utility packages that Shadcn components will use:

```bash
# Install utility dependencies that Shadcn/ui components need
npm install clsx tailwind-merge class-variance-authority

# Install the Tailwind CSS animate plugin (required by many Shadcn components)
npm install -D tailwindcss-animate
```

**What these packages do:**
- `clsx`: Utility for constructing className strings conditionally
- `tailwind-merge`: Intelligently merges Tailwind CSS classes without conflicts
- `class-variance-authority`: Creates component variants with TypeScript support
- `tailwindcss-animate`: Provides animation utilities used by Shadcn components

### Step 2: Initialize Shadcn/ui

Now run the Shadcn/ui initialization command. You'll be prompted with several questions:

```bash
npx shadcn@latest init
```

**Interactive Prompts and Recommended Answers:**

1. **Which style would you like to use?**
   - Select: `Default`
   - Why: Provides the most flexibility for customization

2. **Which color would you like to use as the base color?**
   - Select: `Slate`
   - Why: Neutral base that works well with our custom primary colors

3. **Would you like to use CSS variables for colors?**
   - Select: `Yes`
   - Why: Allows dynamic theming and easier color management

4. **Where is your global CSS file?**
   - Enter: `src/index.css`
   - This is the default location in Vite projects

5. **Would you like to use a custom tailwind prefix?**
   - Select: `No` (just press Enter)
   - Why: We're not using prefixes in our setup

6. **Where is your tailwind.config.js located?**
   - Enter: `tailwind.config.js`
   - This is at the root of the project

7. **Configure the import alias for components?**
   - Enter: `@/components`
   - This creates clean import paths

8. **Configure the import alias for utils?**
   - Enter: `@/lib/utils`
   - Standard location for utility functions

9. **Are you using React Server Components?**
   - Select: `No`
   - Why: We're using Vite, not Next.js with RSC

10. **Write configuration to components.json?**
    - Select: `Yes`
    - This saves your configuration for future component additions

**Alternative: Non-Interactive Installation**

If you prefer to skip the prompts, use this command with all flags:

```bash
npx shadcn@latest init \
  --style default \
  --base-color slate \
  --css-variables \
  --css src/index.css \
  --tailwind-config tailwind.config.js \
  --components "@/components" \
  --utils "@/lib/utils" \
  --no-rsc \
  --yes
```

### Step 6: Configure Path Aliases in Vite

Update your vite.config.ts to support the @/ import alias:

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react-swc'
import path from 'node:path'

// https://vite.dev/config/
export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
})
```

**Note**: If TypeScript complains about the 'path' module:
```bash
# Install Node types if not already installed
npm install -D @types/node
```

## H. Troubleshooting

### Common Issues and Solutions

#### 1. Path Alias Not Working

**Problem**: Imports like `@/components/ui/button` show errors

**Solution**:
```bash
# Restart TypeScript server in VS Code
# Press Ctrl+Shift+P (Cmd+Shift+P on Mac)
# Type: "TypeScript: Restart TS Server"

# Also restart the dev server
npm run dev
```

#### 2. Components Not Found

**Problem**: "Module not found" errors for Shadcn components

**Solution**:
```bash
# Check if component was installed
ls src/components/ui/

# If missing, reinstall the component
npx shadcn@latest add [component-name]
```

#### 3. CSS Variables Conflict

**Problem**: Colors don't match our theme

**Solution**:
```css
/* In src/index.css, ensure our custom values override defaults */
@layer base {
  :root {
    /* Use our specific primary color */
    --primary: 221 83% 53% !important;
  }
}
```

#### 4. TypeScript Errors on Imports

**Problem**: TypeScript can't resolve @/ imports

**Solution**:
```json
// Ensure both tsconfig.json AND tsconfig.app.json have:
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

#### 5. Vite Not Resolving @/ Paths

**Problem**: Runtime errors about module resolution

**Solution**:
```typescript
// vite.config.ts must have:
import path from 'node:path'

export default defineConfig({
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
})
```

#### 6. Components Not Inheriting Theme

**Problem**: Components use default colors, not our theme

**Solution**:
1. Check that CSS variables are defined in src/index.css
2. Ensure Tailwind config extends the theme properly
3. Clear browser cache and hard refresh (Ctrl+Shift+R)

#### 7. Dialog/Sheet Not Opening

**Problem**: Modal components don't appear when triggered

**Solution**:
```tsx
// Ensure you're using the Trigger component correctly
<Dialog>
  <DialogTrigger asChild>
    <Button>Open</Button>
  </DialogTrigger>
  <DialogContent>
    Content here
  </DialogContent>
</Dialog>
```

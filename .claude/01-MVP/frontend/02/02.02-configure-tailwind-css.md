### Step 2: Initialize Tailwind Configuration

**Purpose**: Generate Tailwind and PostCSS configuration files with proper defaults.

**Run initialization command**:
```bash
npx tailwindcss init -p
```

**What this creates**:
1. `tailwind.config.js` - Main Tailwind configuration
2. `postcss.config.js` - PostCSS configuration with Tailwind plugin

**Files created**:
- `/Users/samosipov/Downloads/learn-greek-easy/learn-greek-easy-frontend/tailwind.config.js`
- `/Users/samosipov/Downloads/learn-greek-easy/learn-greek-easy-frontend/postcss.config.js`

**Default tailwind.config.js structure**:
```javascript
/** @type {import('tailwindcss').Config} */
export default {
  content: [],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

---

### Step 3: Configure Tailwind Content Paths

**Purpose**: Tell Tailwind which files to scan for class names to enable tree-shaking.

**Update tailwind.config.js** with content paths:
```javascript
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

**Why this matters**:
- Tailwind scans these files for class names
- Only includes CSS for classes actually used
- Dramatically reduces final CSS bundle size
- Must include all files that use Tailwind classes

---

### Step 4: Import Tailwind Directives

**Purpose**: Add Tailwind's base, components, and utilities layers to your CSS.

**Replace the contents of src/index.css**:
```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

**Remove any existing styles** that might conflict. The file should ONLY contain the three directives above initially.

**What each directive does**:
- `@tailwind base`: Resets and normalizes browser styles
- `@tailwind components`: Adds component classes (if any defined)
- `@tailwind utilities`: Adds all utility classes

**Color mapping from Style Guide**:

- All text variations mapped under `text-*`
- Background colors under `bg-*`

## I. Troubleshooting Guide

### Common Issues and Solutions

#### 1. Styles Not Applying
**Problem**: Tailwind classes don't work
**Solutions**:
- Verify content paths in tailwind.config.js include all your files
- Ensure index.css is imported in main.tsx
- Check that @tailwind directives are in index.css
- Restart dev server after config changes
- Clear browser cache

#### 2. Content Paths Incorrect
**Problem**: Some components don't get styled
**Solutions**:
```javascript
// Make sure content paths match your file structure
content: [
  "./index.html",
  "./src/**/*.{js,ts,jsx,tsx}", // This should catch all React files
]
```

#### 3. CSS Not Reloading
**Problem**: Changes don't appear after saving
**Solutions**:
- Stop dev server (Ctrl+C) and restart with `npm run dev`
- Check that Vite HMR is enabled
- Verify postcss.config.js is properly configured
- Try deleting node_modules/.vite cache folder

#### 4. Custom Colors Not Recognized
**Problem**: Classes like `bg-primary` don't work
**Solutions**:
- Verify color is defined in theme.extend.colors
- Use proper nested structure for color variants
- Check for typos in color names
- Restart dev server after adding new colors

#### 5. PostCSS Errors
**Problem**: Build fails with PostCSS errors
**Solutions**:
- Ensure postcss.config.js uses ES modules syntax (export default)
- Verify all PostCSS plugins are installed
- Check for syntax errors in tailwind.config.js
- Try removing and reinstalling node_modules

#### 6. IntelliSense Not Working
**Problem**: VS Code doesn't autocomplete Tailwind classes
**Solutions**:
- Install Tailwind CSS IntelliSense extension
- Restart VS Code
- Ensure tailwind.config.js is in project root
- Add to VS Code settings.json:
```json
{
  "tailwindCSS.includeLanguages": {
    "javascript": "javascript",
    "typescript": "typescript",
    "javascriptreact": "javascriptreact",
    "typescriptreact": "typescriptreact"
  }
}
```

#### 7. Build Size Too Large
**Problem**: Production CSS file is huge
**Solutions**:
- Verify content paths are specific enough
- Check that unused styles are being purged
- Run production build: `npm run build`
- Analyze build size in dist folder

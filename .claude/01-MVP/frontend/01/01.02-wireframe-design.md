## Overview

This wireframe demonstrates the structure of the main dashboard page that authenticated users see after login. The design follows a dual-purpose approach: primarily functioning as a progress dashboard while secondarily providing access to deck selection. The layout emphasizes clarity, minimal cognitive load, and a warm, encouraging tone suitable for busy adults preparing for naturalization exams.

---

## Layout Structure

### Desktop Layout (1024px+)
- **Fixed Header**: Navigation bar with logo and user menu
- **Main Content Area**: Two-column layout
  - Left column (70%): Dashboard metrics and deck list
  - Right column (30%): Quick actions and secondary information
- **Responsive Grid**: 12-column grid system for flexibility

### Mobile Layout (<768px)
- **Sticky Header**: Compressed navigation with hamburger menu
- **Single Column**: Stacked layout with metrics first, then decks
- **Bottom Navigation**: Key actions accessible via fixed bottom bar

## Component Breakdown

#### Navigation & Layout
- **NavigationMenu**: Main desktop navigation bar

- **Tabs**: For potential section switching (if needed)
- **Separator**: Visual dividers between sections

#### Display Components
- **Card**: For metric cards, deck cards, and action panels
- **Progress**: For deck progress bars
- **Badge**: For deck status indicators
- **Avatar**: User profile image/initials

#### Interactive Components
- **Button**: Various button styles (primary, secondary, ghost)
- **DropdownMenu**: User menu dropdown
- **Dialog**: For confirmation dialogs (start review, etc.)
- **Tooltip**: Hover information for metrics

#### Feedback Components
- **Alert**: Success/error messages
- **Toast**: Temporary notifications
- **Skeleton**: Loading states for content

#### Data Display
- **Table**: For detailed statistics (if expanded)
- **ScrollArea**: For long deck lists

### Custom Components to Build
1. **MetricCard**: Reusable component for dashboard metrics
2. **DeckCard**: Card component with progress bar and stats
3. **QuickActionPanel**: Grouped action buttons panel
4. **MobileBottomNav**: Custom bottom navigation for mobile

## Responsive Considerations

### Desktop (1024px+)
- Three-column grid for metrics (5 cards in a row)
- Two-column layout for main content (70/30 split)
- Full navigation in header
- Hover states on all interactive elements
- Side-by-side layout for content and actions

### Tablet (768px - 1023px)
- Two-column grid for metrics
- Single column for deck cards
- Condensed navigation
- Actions panel moves below decks

### Mobile (<768px)
- Single column throughout
- Sticky header with hamburger menu
- Metrics in 2x2 grid or stacked
- Fixed bottom navigation bar
- Touch-friendly tap targets (min 44px)
- Swipe gestures for deck cards (optional)

### Breakpoint Strategy
```
- Mobile: 0 - 767px
- Tablet: 768px - 1023px
- Desktop: 1024px+
- Wide: 1440px+ (optional container max-width)
```

## Additional Notes

### Design Rationale
- **Metrics First**: Users see their progress immediately, providing motivation and context
- **Clean Layout**: Minimal design reduces cognitive load for busy adults
- **Warm Tone**: Encouraging messages and soft colors reduce exam anxiety
- **Progressive Disclosure**: Detailed stats available but not overwhelming
- **Mobile-First**: Optimized for quick mobile sessions during commutes

### Accessibility Considerations
- High contrast ratios for text
- Clear focus indicators
- Semantic HTML structure
- ARIA labels for screen readers
- Keyboard navigation support

### Performance Optimizations
- Lazy load deck cards below the fold
- Optimize metric calculations
- Use React.memo for expensive components
- Implement virtual scrolling for long deck lists
- Cache dashboard data appropriately

## What "High-Fidelity" Means for HTML Mockup

### Definition
A high-fidelity HTML mockup is indistinguishable from the final production application in terms of:
- **Visual Design**: Pixel-perfect implementation of colors, typography, spacing
- **Interactivity**: All hover states, transitions, and micro-interactions work
- **Content**: Real, meaningful content instead of Lorem Ipsum or generic placeholders
- **Polish**: Smooth animations, loading states, empty states, error states
- **Responsiveness**: Perfect behavior across all device sizes
- **Accessibility**: Keyboard navigation, focus states, ARIA labels

### Not Required for HTML Mockup
- Backend functionality (API calls, data persistence)
- Multi-page navigation (single page is sufficient)
- User authentication flow
- Actual spaced repetition logic

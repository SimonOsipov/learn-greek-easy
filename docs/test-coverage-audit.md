# Test Coverage Audit — Greekly

> **Scope & method.** This audit is **static**: it maps source files to their corresponding test files and reads the code to reason about untested branches and blast radius. It is **not** a runtime coverage-percentage run — no instrumentation, no `--cov` numbers, no executed-line metrics. Findings reflect risk-weighted reasoning about logic that is currently unexercised, not measured coverage gaps.

## 1. Executive Summary

- **27 gaps identified across 3 tiers:** 14 High-risk, 32 Medium-risk, 11 Low-risk (5 of the High findings are distinct exception clusters within the single file `core/exceptions.py`).
- **The backend persistence + core-contract layer is the riskiest cluster.** Every High backend finding sits on a path that touches money or identity: the Stripe webhook idempotency gate (`repositories/webhook_event.py`), the auth-to-user bridge (`repositories/user.py`), the universal write/read base (`repositories/base.py`), the single DB-session factory (`db/session.py`), and the paywall/billing exception contracts (`core/exceptions.py`). A regression in any of these misfires silently — double-billing, cross-user leakage, or app-wide error-response corruption — with no unit test to catch it.
- **The frontend's riskiest gaps are pure-logic transforms and state machines, not components.** Analytics math (`features/analytics/lib/transform.ts`), SRS/progress calculators (`lib/progressUtils.ts`), the culture-session store (`stores/cultureSessionStore.ts`), the SSE audio hook (`useGenerateAudio.ts`), and the exam timer (`useMockExamTimer.ts`) all contain branchy arithmetic or multi-state lifecycles that are mocked out at the page level, leaving their own logic with zero exercised paths.
- **Systemic pattern — "tested through mocks, never directly."** A recurring theme across all tiers: the unit under audit is stubbed by its callers' tests (page mocks the hook, orchestration test mocks `db.execute`, component mocks the context/store), so the integration test passes while the real logic is never run. This is the single most common reason a gap exists.
- **Auth and payments are the highest-stakes subsystems.** The entire auth lifecycle (`LoginForm`, `RouteGuard`, `ResetPassword`, `ForgotPassword`, `Callback`, `repositories/user.py`) and the entire billing-exception surface are High-risk and largely untested at the unit level — a regression here locks users out or charges them wrong.

- `learn-greek-easy-frontend/src/stores/adminTabCountsStore.ts` — `tabCountsFetchSeq` monotonic counter discards stale responses; cosmetic badge counts only.
- `learn-greek-easy-frontend/src/lib/cefrColors.ts` — Lookup-table functions with silent fallbacks; wide blast radius (every deck badge) but minimal logic — worth a pinning test.
- `learn-greek-easy-frontend/src/utils/audioSpeed.ts` — Single binary guard ('0.75' else 1) + SSR/localStorage guards; failure mode is always a safe default.

1. **`learn-greek-easy-backend/src/repositories/webhook_event.py`** — The idempotency gate for real money. An untested state-machine regression here means double-billing live users; highest blast radius × highest stakes.
2. **`learn-greek-easy-backend/src/core/exceptions.py`** — One file, three High clusters (BaseAPIException root, PremiumRequired paywall, billing cluster). Pure-unit tests, no fixtures, and it underpins every API error response and the entire paywall — best risk-reduction-per-effort in the codebase.
3. **`learn-greek-easy-backend/src/repositories/user.py`** — The auth-to-user bridge on every authenticated request; a bug denies valid users or leaks across accounts. Identity correctness must be pinned.
4. **`learn-greek-easy-backend/src/repositories/base.py`** — Inherited by every repository; `exclude_unset` write semantics and `get_or_404` are silently load-bearing app-wide. Testing the base protects all subclasses at once.
5. **`learn-greek-easy-frontend/src/components/auth/LoginForm.tsx`** (with `RouteGuard.tsx`) — Security-critical role derivation and the auth lifecycle; a regression locks all users out of the app. Highest-stakes frontend path.
6. **`learn-greek-easy-frontend/src/lib/progressUtils.ts`** + **`features/analytics/lib/transform.ts`** — Pure, dependency-free calculation modules behind everything learners see about their progress; cheap to test, currently mocked away, and silently misreport mastery/streaks/accuracy.
7. **`learn-greek-easy-frontend/src/stores/cultureSessionStore.ts`** — The runtime backbone of the culture feature with a cross-user recovery guard; the most complex untested store, with a data-leak failure mode.
8. **`learn-greek-easy-backend/src/schemas/review.py`** — Guards the SRS quality value that drives SM-2 scheduling; a quick unit test pins the validator and adds a regression guard so the second gate doesn't silently rot as the enum evolves.
